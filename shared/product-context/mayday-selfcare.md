# Mayday Selfcare — Module Context

**Baseline** : Autonomisez vos clients dans leurs recherches. **Key metric** : −40% incoming tickets.

Customer-facing self-service portal. Exposes a public knowledge base so end customers find answers autonomously. Same content source as Mayday Knowledge — different access layer.

## Interfaces
- **Mayday Admin** — KM platform for creating and managing Selfcare content and settings
- **Selfcare Portal** — branded public knowledge base for end customers; multiple portals supported, each independently configurable
- **Selfcare Widget** — embeddable interface integrated into any website, product, or customer journey

## Key Features
- **Content formats** — same as Knowledge: Article and Diagnostic/Guide
- **Content management** — same status lifecycle as Knowledge (Draft → Published → Archived); content labels for organization and filtering
- **Shared Elements** — reusable blocks from Knowledge available on Selfcare portals
- **Customization** — branding, custom homepage images, URL redirections, multilingual (16+ languages), SEO optimization
- **Ask Mayday** — AI-powered assistant on the portal; customizable per portal (tone, scope, sources); supports exact match search
- **Plugins** — similar content recommendations, contact form, satisfaction/feedback collection, breadcrumb navigation, content label filtering
- **Analytics** — consultation tracking, language-based breakdown, group filtering, dashboard
- **Collaboration** — multiple KMs can collaborate on the same Selfcare knowledge base

## Key Distinction
Selfcare is a publication layer on top of Knowledge — content is authored once in Knowledge and selectively exposed on Selfcare. KMs control what is public.
